# 自媒体增长驾驶舱路线图

路线图只定义经过当前实例验证后值得继续产品化的闭环，不以“能开发”为新增功能的理由。

## v0.2.0：可配置的本地增长驾驶舱

- 使用者、创作定位、增长计划、日期、目标和平台账号配置化。
- 今日任务、行动目标、内容、作品复盘、账号拆解、每日复盘与 Obsidian 受控双向写回。
- 空 Vault 初始化、脱敏测试数据和持续集成门禁。
- 本地单用户，不包含登录、云端内容同步或平台数据自动抓取。

## v0.3.0：每日经营闭环

下一版本只解决一个主问题：让当天事实进入每日复盘，再把确认后的明日动作送入第二天任务。

验收标准：

1. 新建每日复盘时，自动汇总当天任务、发布、粉丝变化、作品复盘和账号拆解等客观事实。
2. 自动事实与人工判断分区，系统不替使用者编造结论。
3. 已确认复盘可将“明日最重要动作”一键加入下一天三件事，并保留来源链接。
4. 写回 Obsidian 后，网页与原文状态一致；冲突时不静默覆盖。

## v0.4.0：本机 AI 协作工作台

在 v0.2 的内容与数据闭环上增加一个受控的执行层：从真实内容资产发起任务，通过 ACP 调用本机 Codex、Claude Code、Kimi Code、Gemini CLI 或 Grok Build，人工确认敏感动作和最终写回。

验收标准：

1. 五个 Agent 的安装、版本、ACP 与登录状态可被独立检测，驾驶舱不自动安装、升级或读取凭证。
2. 每次运行使用独立临时工作区，Agent 不直接读写整个 Vault。
3. 回复、计划、工具调用、权限请求、文件差异和结束状态通过结构化事件呈现。
4. 敏感动作逐次批准或拒绝；首版不提供永久授权和 YOLO。
5. 人工确认前 Obsidian 不变化；确认后通过现有安全写入链路保存结果。
6. Mock ACP 异常矩阵和五个本机 CLI 最小只读联调通过。

详细边界见 [docs/V0.4-AI-COLLABORATION.md](./docs/V0.4-AI-COLLABORATION.md)。多 Agent 自动编排不进入 v0.4。

## v0.5.0：AI 任务成果闭环

当前版本不继续增加模型或通用聊天能力，只打通一条真实业务链路：从“今日三件事”把任务交给本机 AI，把人工确认后的结果新建为文章草稿、短视频草稿、复盘草稿或次日任务。

验收标准：

1. 今日任务与选题、内容、复盘等权威资产的关系可以写入 Obsidian 并稳定回读。
2. 从今日任务进入 AI 工作台时，服务端重新解析真实任务与资产，浏览器不能伪造来源正文或路径。
3. 人工确认前 Vault 不变化；确认后只新建预览中的业务成果，不覆盖已有资产。
4. 文章、视频和复盘草稿不自动计入完成数，AI 交付也不自动勾选原任务。
5. 重复确认保持幂等；来源变化返回冲突；交付状态不确定时保留可核对成果并允许同一请求安全认领，不重复生成。
6. 至少完成一次真实本机 Agent 的“任务 → 运行 → 预览 → 写回 → 页面回流”端到端验证。

详细契约见 [docs/V0.5-AI-TASK-LOOP.md](./docs/V0.5-AI-TASK-LOOP.md)。多 Agent 自动编排、通用聊天历史、自动发布和无人值守任务不进入 v0.5。

## v0.6.0：交互式 AI 协作工作台

把 v0.5 的一次性运行升级为本机 CLI 的长期多轮会话。网页只承担结构化对话、权限、接受与导入交互，不模拟终端，也不把浏览器刷新当作会话结束。

验收标准：

1. 空工作台可直接发送自由消息；资产上下文为可选项，不再要求先选择模板。
2. 同一 Conversation 可以连续多轮沟通；页面重新读取和本地服务重启后，历史记录仍在，支持恢复的 Provider 继续原 Session。
3. 每轮回复、计划和工具事件先持久化，再以完整 Conversation 快照通过 SSE 回流页面，不使用秒级轮询。
4. “接受本轮”只记录人工选择，不写入 Vault；再次确认导入后，只在项目 `03-工作过程/AI协作` 新建一份权威成果。
5. 导入不关闭 Conversation；用户可以继续追问，显式关闭才回收本产品启动的 Agent 进程。
6. 隔离临时 Vault 中的真实 Codex 验证覆盖三轮上下文、服务重启恢复、写入边界、导入后继续和进程回收。

详细合同见 [docs/V0.6-INTERACTIVE-AI-WORKBENCH.md](./docs/V0.6-INTERACTIVE-AI-WORKBENCH.md)。V0.6 只对完成真实专项验收的 Provider 宣称“已验证”；其余已安装 CLI 继续显示检测结果，不用一次握手冒充长期会话认证。

## v0.6.1：本机 AI 环境中心与 Antigravity

在不打断对话主界面的前提下，把“能否使用本机 AI”变成一个可检测、可选择处理、可复验的闭环。

验收标准：

1. 工作台检测 Codex、Claude Code、Kimi Code、Antigravity 和 Grok Build；可用 Provider 直接进入新会话下拉框。
2. 缺失、过期和待登录状态只在紧凑管理抽屉中提供相应动作，不在聊天区堆放安装说明。
3. 安装、更新和登录均由用户点击并确认；服务端使用固定官方白名单，浏览器不能提交命令、包名、URL、参数或路径。
4. 操作结束后重新探测真实 CLI；安装未生效或版本未变化时明确失败。
5. 新界面使用 Antigravity；旧 `gemini` 仅保留历史数据可读性。
6. Antigravity 在 `plan + sandbox` 下完成真实两轮会话和上下文恢复；首版不开放写入权限。

详细边界见 [docs/V0.6.1-AI-ENVIRONMENT-CENTER.md](./docs/V0.6.1-AI-ENVIRONMENT-CENTER.md)。

## 后续候选：首次设置产品化

只有在另一位真实使用者完成一次命令行初始化，并证明安装门槛阻碍使用后，才开发图形化首次设置。候选范围包括 Vault 选择、平台增删、基线录入和连接检查；不提前加入多用户、账号登录、云端内容库或自动发布。
