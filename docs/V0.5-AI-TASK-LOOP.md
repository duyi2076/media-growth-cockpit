# V0.5 AI 任务成果闭环

## 产品目标

V0.5 只解决一个问题：让“今日三件事”中的一项真实任务，经过一次本机 AI 协作后，形成能够进入下一业务环节的成果，并在人工确认后受控写入 Obsidian。

V0.4 已经验证了五类本机 AI CLI 的发现、调用、事件展示、权限确认与通用结果保存。V0.5 不继续扩张成通用聊天产品，而是补齐驾驶舱的业务闭环：

```text
今日任务
  ↓ 交给 AI
关联真实资产
  ↓
单个本机 Agent 执行
  ↓
形成结构化成果提案
  ↓
预览目标位置、来源和正文
  ↓ 人工确认
新建衍生资产或次日任务
  ↓
Obsidian 与页面同步
  ↓
使用者自己确认原任务是否完成
```

## 首版使用场景

### 场景一：今日任务生成文章或短视频草稿

1. 使用者在“今日三件事”中选择一项任务并点击“交给 AI”。
2. 若任务已经关联选题、内容或复盘，系统自动带入权威原文；未关联时必须由使用者选择，AI 不猜来源。
3. 使用者选择文章草稿或短视频草稿模板，再选择一个本机 Agent。
4. Agent 完成后，页面展示原始结果和“生成内容草稿”入口。
5. 页面预览新资产类型、目标目录、标题、来源关系和正文。
6. 使用者确认后才创建新文件；原任务只显示“AI 已交付”，不会自动勾选完成。

### 场景二：今日任务生成复盘草稿

1. 使用者从今日任务进入 AI 工作台，并关联已发布内容或对标账号素材。
2. Agent 返回分析结果。
3. 使用者选择“生成复盘草稿”，可补充摘要和下一步动作；核心发现固定采用 Agent 的最终结果。
4. 确认后创建待人工确认的内容复盘或账号拆解。
5. 草稿进入“复盘与对标”，但不计入已完成复盘数量。

### 场景三：复盘生成次日任务

1. 今日任务关联一份已确认的复盘。
2. Agent 提出次日行动建议。
3. 使用者在预览中保留一至三项具体动作。
4. 确认后写入服务器计算出的下一自然日任务文件。
5. 今天的任务状态不被自动修改。

## P0 范围

### 1. 任务与资产关系必须持久化

今日任务仍使用现有 Markdown 复选框作为正文，但关系单独保存在同一日文件的 Frontmatter 中：

```yaml
task_links:
  task-20260714-a1b2c3:
    type: content
    id: content-20260713-demo
```

允许的关联类型：

- `topic`
- `content`
- `content-review`
- `account-breakdown`
- `daily-review`
- 兼容已有项目任务的 `task`

关系以任务 ID 为键；任务删除时同步移除孤立关系。浏览器不能提交文件路径。

### 2. AI 运行记录来源任务

从今日任务发起的运行，在权威 manifest 中增加：

```yaml
sourceTask:
  id:
  date:
  title:
  linkType:
  linkId:
  fingerprint:
```

服务端根据 `sourceTaskId` 重新读取当天任务和关联资产。标题、资产类型、资产 ID、原文路径与哈希均来自服务端，不信任浏览器摘要。

### 3. 只支持三种业务成果

```text
content_draft  → 文章草稿或短视频草稿
review_draft   → 内容复盘或账号拆解草稿
next_day_task  → 下一自然日的一至三项任务
```

每次运行首版最多创建一个业务成果。通用“保存 AI 结果”继续保留，但不等同于业务交付。

### 4. 写回前必须预览

确认页必须展示：

- 成果类型。
- 新文件或新任务的目标位置。
- 来源任务和来源资产。
- 将写入的标题与正文。
- 明确的“确认写入”动作。

打开预览不会修改 Vault。取消预览不会留下临时业务文件。

### 5. 写回后保持业务语义

- AI 交付不自动勾选今日任务。
- 内容草稿没有 `completed_at`，不计入文章、视频或发布完成数。
- 复盘草稿保持 `confirmation: 待人工确认`，不计入已完成复盘。
- 次日任务默认未完成。
- 今日任务通过运行记录显示“AI 已交付”和成果入口，不重复写一份状态到日任务正文。

## 权威数据契约

### 今日任务

```ts
type TodayTask = {
  id: string
  title: string
  done: boolean
  linkType:
    | "topic"
    | "content"
    | "content-review"
    | "account-breakdown"
    | "daily-review"
    | "task"
    | null
  linkId: string | null
}
```

### AI 来源任务

```ts
type AiSourceTask = {
  id: string
  date: string
  title: string
  linkType: Exclude<TodayTask["linkType"], null>
  linkId: string
  fingerprint: string
}
```

`fingerprint` 只覆盖当前任务的 ID、标题和关联关系。它不使用整份日任务文件哈希，避免同日其他任务变化造成无关冲突。

### AI 业务交付

```ts
type AiDelivery = {
  id: string
  kind: "content_draft" | "review_draft" | "next_day_task"
  status: "completed"
  sourceRunId: string
  sourceTaskId: string
  targetType: "content" | "review" | "task"
  targetId: string | null
  targetRelativePath: string
  targetTitle: string
  createdAt: string
}
```

成果文件还必须保存：

```yaml
source_run:
source_task_id:
derived_from:
related_assets:
```

## HTTP 接口

### 任务关系

- 继续使用 `PUT /api/daily-tasks/today` 保存今日任务及其关联关系。
- 服务端校验 `linkType` 与 `linkId` 必须同时为空或同时存在。
- 构建索引时从 Frontmatter 恢复关系，不能再把关联字段丢弃。

### 发起 AI 运行

`POST /api/ai-runs` 增加可选字段：

```json
{
  "sourceTaskId": "task-20260714-a1b2c3"
}
```

从任务发起时，`context` 仍用于声明期望关联对象，但服务端必须验证它与任务保存的关系完全一致。

### 创建业务成果

```text
POST /api/ai-runs/:runId/deliveries
```

请求只允许业务字段，不能提交任意文件路径、原始来源路径或替换正文：

```json
{
  "kind": "content_draft",
  "contentFormat": "文章",
  "title": "标题"
}
```

`contentFormat` 只允许 `文章` 或 `短视频口播`；页面可以显示更短的“短视频”，但 HTTP 领域值保持统一。

```json
{
  "kind": "review_draft",
  "reviewKind": "content-review",
  "title": "标题",
  "summary": "摘要",
  "nextAction": "下一步"
}
```

```json
{
  "kind": "next_day_task",
  "tasks": ["动作一", "动作二"]
}
```

内容正文和复盘核心发现都只来源于服务端保存的 `finalText`；摘要与下一步是可选的人工业务字段。次日日期由服务端根据本地时区计算，浏览器不能指定任意日期。

## 一致性、幂等与冲突

### 确认前

- 不创建目标文件。
- 不修改任务状态。
- 不修改行动完成数。

### 确认时

1. 运行必须为 `completed`，并且有非空最终结果。
2. 重新读取来源任务，比较任务指纹。
3. 重新读取关联资产，比较运行开始时保存的 SHA-256。
4. 任一来源变化返回 `409 Conflict`，要求重新运行或人工确认新上下文。
5. 使用固定目录、服务器生成 ID、原子写入、备份和写后索引校验。
6. 运行清单未记录或暂时不可读时不删除新成果；保留 `source_run`、请求哈希和载荷哈希，由相同请求安全认领。

### 重试

- 同一运行、同一成果请求重复提交返回已创建成果，不重复写入。
- 同一运行不能创建第二种业务成果；需要新成果时重新发起运行。
- 运行 manifest 是交付关系的权威记录；SQLite 只用于查询，失败不能破坏权威文件。
- 若进程恰在普通网页新建文件落盘后、审计状态写入前崩溃，文件会保留供人工核对，不会被自动删除或自动认领。

## 页面设计

### 增长总览

今日任务每项增加一个轻量操作：

```text
交给 AI
```

- 已关联资产时直接进入 `/ai?taskId={id}`。
- 未关联时进入 AI 工作台并要求先选择关联资产。
- 已交付时显示“AI 已交付”，点击可查看运行和成果。
- 任务复选框仍由使用者自己确认。

### AI 工作台

页面在现有布局上增加三个状态区：

1. **来源任务**：显示任务、日期与关联资产。
2. **本次执行**：保留 Agent、模板、要求、流式事件和权限控制。
3. **成果交付**：完成后选择成果类型，预览并确认写回。

不增加聊天历史侧栏、不增加多 Agent 编排画布、不增加终端输入框。

### 内容工作台与复盘页面

- 新内容草稿通过现有索引立即出现。
- 新复盘草稿显示“待人工确认”，沿用既有确认流程。
- 页面只显示对创作决策有用的来源关系，不显示本机路径、哈希、运行目录或调试字段。

## 安全边界

- 本版本不扩大 V0.4 的 CLI 权限和进程边界。
- 浏览器只能选择服务端索引中的资产 ID。
- 浏览器不能指定 Vault 路径、可执行文件、CLI 参数、环境变量或来源正文。
- 成果只允许新建在白名单目录，禁止覆盖已有资产。
- 路径穿越、软链接逃逸、异常 Frontmatter 与超限正文均拒绝。
- `sensitivity` 继续影响索引与上下文复制；敏感资产不能进入 AI 临时任务空间。
- 服务仍只绑定回环地址，不开放公网或局域网。

## 明确不做

- 多 Agent 自动分工、并行讨论、互相调用或自动复核。
- 通用聊天历史、分支对话、长期 Agent 记忆。
- 网页终端和整个 Vault 的自由检索。
- AI 自动发布、自动勾选任务、自动增加行动完成数。
- 覆盖已有文章、复盘、知识或业务资产。
- 自动安装、升级、登录或切换本机 CLI。
- 定时运行、无人值守任务、云端多用户和远程控制。

## 发布验收

### 自动化

1. 今日任务关联关系写入、读取、删除和索引回读一致。
2. 浏览器伪造任务标题、资产摘要、路径或关系均被服务端拒绝。
3. 文章稿、视频稿、复盘稿和次日任务分别完成服务端集成测试。
4. 确认前 Vault 哈希不变；确认后只有预告目标发生变化。
5. `source_task_id → source_run → derived_asset` 可追溯。
6. 草稿不增加文章、视频、发布或复盘完成数。
7. 重复确认保持幂等；来源变化返回 409；清单状态不确定时返回可重试状态，成果不丢失也不重复生成。
8. Obsidian 修改通过现有索引与 SSE 回流页面。
9. 前端、服务端、索引、类型、构建、数据校验与依赖安全测试全部通过。

### 真实流程

1. 从真实今日任务生成一篇文章草稿并写回。
2. 从真实今日任务生成一条短视频草稿并写回。
3. 从真实复盘生成下一自然日任务。
4. 在 1280×720 与 1440×900 桌面视口完成主流程，控制台错误和警告为 0。
5. 五个本机 Agent 不重复做全量测试；至少用一个可用 Agent 完成真实端到端业务交付，并复用 V0.4 的 Provider 回归测试。

### 使用证据

连续使用三天并完成至少五个真实 AI 任务。至少四次无需复制粘贴即可进入下一业务环节，才证明闭环有效；至少出现两次真实修订需求，才进入 P1 多轮修订设计。
