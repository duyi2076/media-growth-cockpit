# 自媒体增长驾驶舱 2.0：设计与产品验收

## 验收对象

- 产品：自媒体增长驾驶舱 2.0
- 地址：`http://127.0.0.1:4173/`
- 产品类型：桌面浏览器中的本地工作台
- 核心任务：把使用者自己的增长目标、内容生产、复盘对标和第二大脑资产连接成一个日常操作界面
- 数据边界：`V2_VAULT_ROOT` 指向的 Obsidian V2 是权威资产库；网页只对已开放的白名单字段进行受控写回

## 视觉真源与最终截图

视觉参考和验收截图只保存在本机，不进入代码仓库。`qa/` 已被版本控制排除，避免账号信息、内容画面和本机路径进入远端历史。仓库仅保留可复用的验收结论。

## 最终信息架构

一级导航只保留六个使用场景：

1. 增长总览：回答“目标差多少、今天做什么、动作完成到哪里”。
2. 内容工作台：回答“每份内容现在在哪一步、下一步是什么”。
3. 复盘与对标：回答“哪些内容需要复盘、哪些账号已经拆解”。
4. 每日复盘：回答“今天完成了什么、形成了什么判断、明天最重要的动作是什么”。
5. AI 协作：回答“选中一份真实资料后，哪个本机 AI 能完成当前任务，结果是否值得保存”。
6. 资产检索：回答“第二大脑里已有的内容、知识和项目资产在哪里”。

素材库、选题 Lab、实验解释、来源路径和数据状态不再占用一级导航。它们被降为具体操作、筛选视图或后台能力。

## 构图与认知负荷

- 首屏保留净增粉目标、行动目标和今日三件事，符合“先看方向，再看动作”的阅读顺序。
- 已配置平台的当前粉丝数压缩进“目标进度”标题行，不再作为独立模块；数值可直接编辑并写回 V2。
- 内容工作台只保留“选题、发布、复盘”三个阶段；制作过程与“调研中/创作中”状态已经从产品和数据契约中删除。
- 每个内容卡只保留标题、状态、形态、平台、下一步和截止日期；路径、索引说明、数据证据不在主界面展示。
- 详情和编辑进入右侧抽屉，避免第四栏和页面级横向滚动。
- 复盘与账号拆解共用一个模块，通过双标签切换，不再重复导航。
- 资产检索默认排除敏感内容和待确认内容；待确认资产只能由用户主动打开筛选。

## 关键交互验收

- 六个一级路由均能打开，导航选中态与页面标题一致。
- 内容工作台只出现“选题 / 发布 / 复盘”，当前状态筛选中没有“调研中 / 创作中”。
- 历史已发布卡片因缺少完整发布时间和核验记录，明确显示“待核验”。
- 未经核验的历史内容不会进入“内容复盘”；复盘入口只接受已核验发布内容或明确标记为“待复盘”的资产。
- 正式开始前五项行动完成数全部为 0；点击“正式开始统计”后必须再次确认，取消不会写入 V2。
- 文章、视频、发布、复盘、账号拆解五项目标数量均持续显示为可编辑数字输入框；回车或失焦保存，Esc 撤销未保存修改。
- 左侧完成数量继续由 V2 资产统计，不开放人工修改。
- 内容工作台支持按标题、形态、平台和状态筛选。
- 看板和表格视图可切换。
- “新建选题”打开右侧抽屉，保存后写入 V2 的选题池。
- 已有内容可修改状态、形态、发布平台、下一步和截止日期。
- 内容详情可打开对应的 Obsidian 原文。
- “打开原文”已经从会被浏览器拦截的自定义协议链接改为同源本地接口；资产检索实测成功打开 `目标与验收.md`。
- 内容复盘和账号拆解标签可切换。
- 两类复盘都可从网页新建、编辑和确认；确认会写入 Obsidian，已确认资产再次编辑后自动回到待人工确认。
- 每日复盘有独立页面；只有六项完整且经过人工确认的每日复盘才计入“复盘”行动完成数。
- AI 协作可从权威资产选择任务资料，能够启动真实 Codex 任务、持续显示运行结果并保留运行记录；驾驶舱不会在未点击“保存到 Obsidian”时触发受控导入，但当前没有操作系统级沙箱，不能保证失控进程无法主动访问本机文件。
- 权限模式明确区分原生只读/计划模式与 CLI 主动上报的逐次授权，不把任务目录描述成系统沙箱。
- AI 最终回复保持展开；计划、工具调用和文件修改收进“运行详情”，任务完成后默认折叠，避免长任务挤占主要阅读区域。
- 资产检索可跨内容、知识和项目文档搜索，并打开详情或原文。
- 浏览器控制台错误与警告：`0`。

## 双向数据验收

- 今日三件事：网页可新增、编辑、完成；按北京时间写入当天 Obsidian 文件；外部修改通过事件通道即时回流，页面重新获得焦点时补拉。
- 行动目标：网页可填写文章、视频、发布、复盘、账号拆解目标；正式开始时间写回固定目标文件；开始前完成数固定为 0。
- 发布验证：只有开始时间之后、同时包含完整 `published_at`、`verification: 已核验`，以及 HTTPS URL 或 `evidence_ref` 的记录才进入完成数。
- 内容资产：新建选题只写入 `30-内容资产/00-选题池`；既有内容只开放白名单字段；Obsidian 外部修改触发索引重建并回流页面。
- 写入安全：路径与软链接防护、SHA-256 冲突检测、串行队列、同目录原子替换、备份、失败回滚、写后重建与校验均有自动测试。

## 设计系统检查

- 视觉层级：暖灰背景、白色工作面、石墨正文、深青主色和少量语义色；不存在霓虹、渐变或装饰性图标堆叠。
- 字体：使用稳定的中文无衬线字体栈，标题、正文、标签和数字有明确层级。
- 间距：页面、模块、卡片和字段使用一致的紧凑节奏，符合桌面生产力工具的信息密度。
- 卡片：只在“内容条目、任务条目、资产条目”本身就是交互对象时使用，没有说明性卡片拼贴。
- 文案：使用目标、状态和动作语言；页面中没有“只读连接 V2”“演示模式”“来源文件”“数据状态”等开发过程信息。
- 图标：统一使用 Phosphor Icons；没有 emoji、手绘 SVG 或无意义装饰图。
- 可访问性：桌面窄屏导航提供 `aria-label` 和 `title`；主要交互有按钮语义、选中态和键盘关闭逻辑。

## 迭代对照

### 1. 增长总览

- 旧问题：六平台各占一张大卡；右侧存在来源、索引、只读和演示说明；目标下方是与目标关联较弱的内容摘要。
- 2.0：六平台压缩进目标进度标题行；删除独立六平台模块和工作过程说明；目标下方改为文章、视频、发布、复盘、账号拆解五项可编辑行动目标。
- 指标口径修正：启动基线与净增目标分开；当前进度只统计项目开始后的净增粉，不再把初始总粉丝当作已完成进度。
- 最新修改：五项目标数量由“点击铅笔后编辑”改为始终可编辑；移除逐项铅笔、确认和取消图标，保留聚焦边框与自动保存反馈。对照图确认五个输入框同屏、节奏一致、没有新增说明文字。

### 2. 内容工作台

- 旧问题：八个状态横向排列，桌面需要横向滚动；各阶段稀疏；本地模拟记录和真实 V2 资产边界不直观。
- 2.0 第一轮：曾收敛为“选题 / 制作 / 发布 / 复盘”四阶段。
- 最新反馈：制作过程不应成为资产状态，因此删除“制作”栏目以及“调研中 / 创作中”两个状态。
- 修复后：页面只保留三阶段；已发布但证据不足的历史内容显示“待核验”，且不会进入上线后的行动完成数。
- 对照证据：`v2-content-remove-production-before-after.png` 显示被标注的制作列已经消失，剩余三列重新均分宽度，没有空洞或横向滚动。

### 3. 复盘与对标

- 旧问题：复盘、实验、待审核和 Agent 状态分别占用认知入口。
- 2.0：收敛为内容复盘和账号拆解两个真实业务维度；两类资产都可新建、编辑和人工确认，没有证据的账号拆解不生成示例数据。

### 4. 每日复盘

- 旧问题：单条作品复盘与一天整体经营复盘混用同一概念。
- 2.0：每日复盘成为独立页面和独立资产类型；客观事实、有效动作、问题、判断与明日动作分开记录，确认后才计入行动完成数。

### 5. 资产检索

- 旧问题：素材、知识、项目文档和来源文件分散，用户需要先理解第二大脑结构。
- 2.0：以统一搜索入口呈现资产类型、状态和主题；文件路径只在后台使用。
