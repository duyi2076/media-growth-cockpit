# 变更记录

本项目使用 [Semantic Versioning](https://semver.org/) 管理代码版本。当前代码发布版本对应 `v0.6.1`；在安装与迁移流程稳定前，不承诺 `1.0.0` 的长期兼容性。

## [0.6.1] - 2026-07-14

### 新增

- AI 工作台新增紧凑的“管理本机 AI”抽屉，统一检测 Codex、Claude Code、Kimi Code、Antigravity 与 Grok Build 的安装、版本和登录状态。
- 缺失、过期或待登录的 Provider 提供用户点击确认后的安装、更新或登录动作；浏览器只能提交固定 Provider 与动作。
- Antigravity 使用官方 `plan + sandbox + conversation` 合同接入长期多轮会话，并保留旧 `gemini` Provider 的历史数据兼容。

### 调整

- 新会话不再显示旧 Gemini CLI，产品语义统一为 Antigravity。
- 安装和更新完成后重新探测真实 CLI；未安装成功或版本未变化时显示失败，不把脚本退出冒充为成功。
- Antigravity 更新调用已验证本机 CLI 的官方 `update` 子命令，不依赖安装脚本覆盖旧二进制。

### 安全

- 环境管理接口只接受本机回环、同源与 CSRF 请求，服务端固定官方包名、脚本地址、参数和可执行入口。
- 登录使用固定临时 Terminal 脚本；凭证不进入浏览器、任务状态或日志。
- Provider 拥有未关闭会话时拒绝安装或更新；Antigravity 首版只开放只读协作。

## [0.6.0] - 2026-07-14

### 新增

- AI 协作升级为可恢复的长期 Conversation，同一个本机 Agent Session 可以连续处理多个 Turn。
- 空工作台支持直接发送自由消息；现有任务或资产只作为可选上下文，不阻断普通对话。
- 新增会话历史、完整快照 SSE、后续消息、取消、单次权限、接受本轮、确认导入和显式关闭接口。
- 权威资产原文经哈希复验后复制到会话 `workspace/inputs/`；每轮开始前重新核验副本，Agent 工作区与权威会话控制文件相互分离。
- 新增真实 Codex 三轮验证脚本，覆盖服务重启恢复、首轮 nonce 回忆、接受前后 Vault 不变、白名单导入、导入后继续与进程回收。

### 调整

- 页面从“一次提交、一次结果”改为 Claudian 式长期消息区和底部输入框；计划、工具与权限作为结构化折叠信息，不显示终端日志。
- V0.6 Conversation 不复用 V0.4 Run 的 SQLite 查询索引；每个会话以 manifest、session、turn JSON 和有界 JSONL 事件独立持久化。
- SSE 只发送最新完整 Conversation 快照，不向浏览器透传原始 ACP 增量消息。
- 接受结果只记录 `acceptedTurnId`、哈希与确认时间；再次确认导入才创建 Obsidian 成果，导入后会话仍保持打开。
- Provider 状态区分“已安装/可握手”和“已完成 V0.6 长期会话验收”；当前真实发布验收只对 Codex 做保证。

### 安全

- 长期 Agent 使用固定绝对入口、`shell:false`、独立会话工作区和最多两个活动连接；显式关闭回收本产品启动的进程组。
- 会话公开 DTO 隐藏 Provider Session ID、工作目录、可执行文件与环境变量，并对事件、错误和输出做脱敏与大小限制。
- 人工导入只允许原子新建到当前项目 `03-工作过程/AI协作`，拒绝路径穿越、软链接、危险 HTML、疑似凭证、超限正文与确认期间的版本漂移。

## [0.5.0] - 2026-07-14

### 新增

- 今日任务可关联选题、内容、内容复盘、账号拆解或每日复盘，并从任务直接进入 AI 工作台。
- AI 运行记录保存来源任务、来源资产与启动时指纹，服务端在交付前重新验证全部权威来源。
- 新增文章/短视频草稿、复盘/账号拆解草稿、下一自然日任务三类业务交付。
- AI 完成后先展示目标、来源与正文预览；只有人工确认才写入 Obsidian。
- 交付回执可从今日任务和 AI 工作台回看，并可通过受控接口打开 Obsidian 原文。
- 新增真实本机 Agent 验证脚本，覆盖“任务 → Codex → 内容草稿 → Obsidian 回读”。

### 调整

- AI 交付不会自动勾选来源任务，也不会给草稿写入 `completed_at` 或提前增加行动完成数。
- 复盘成果固定进入待人工确认，次日任务固定为未完成；原有确认与计数规则继续作为唯一口径。
- 从今日任务发起的 AI 运行只允许绑定该任务保存的权威资产，浏览器不能替换来源或正文。
- 成果类型根据来源资产收敛：复盘来源只能继续形成次日任务，或形成与来源一致的复盘草稿。
- 当天任务文件尚不存在时，项目建议任务会明确显示“保存为今天任务”；点击 AI 委派会先完成同一份 Obsidian 落盘，保存失败不会进入无来源任务页面。

### 安全

- 业务交付只接受严格业务字段，拒绝浏览器提交路径、正文、发现列表、日期或任意 Frontmatter。
- 创建与幂等恢复均复验目标目录、软链接、inode 与内容哈希，防止路径竞态和误认领人工修改文件。
- 写入后重新读取并核对权威载荷；清单未记录或暂时不可读时保留带完整交付哈希的成果，由相同请求安全认领，不再通过路径删除不确定文件。
- 来源任务指纹、来源资产 SHA-256 与交付请求哈希共同保证重试幂等和来源漂移可见。

## [0.4.0] - 2026-07-14

### 新增

- 新增 AI 协作一级模块，可从选题、内容、复盘、账号拆解和每日复盘发起八类任务。
- 接入本机 Codex、Claude Code、Kimi Code、Gemini CLI 与 Grok Build；本机真实验证四个通过，Gemini 当前被认证方式阻塞。
- 新增五个 CLI 的安装、版本、适配器与协议能力检测，不自动安装、升级、登录或读取凭证。
- 新增独立任务工作区、SQLite 元数据、连续 JSONL 事件和 SSE 实时运行快照。
- 新增回复、计划、工具调用、权限请求、错误、取消和终态展示。
- 新增一次性允许/拒绝权限流程，以及人工二次确认后的 Obsidian 结果导入。
- 浏览器只提交资料类型与资产 ID；服务端从权威索引解析真实 Markdown、来源关系和 SHA-256，再复制到任务工作区。
- 每次运行保存 Provider、ACP 适配器和协议版本证据，导入 Obsidian 时一并记录。

### 调整

- 统一开发与常驻服务运行时为 Node.js 24.14.0、npm 11.9.0，避免原生 SQLite 依赖因 Node 主版本不一致而无法启动。
- AI 任务默认只读；Codex、Claude Code 与 Gemini 使用原生只读/计划会话模式。Kimi 与 Grok 没有可强制的原生只读模式，因此仅开放分析、拒绝写入权限请求，并明确提示这不是系统级隔离。
- 页面使用实时事件流代替秒级轮询，不因后台更新清空任务表单。
- 服务重启会把遗留活动任务明确标为失败，避免页面显示不存在的后台执行。
- 运行结果只在人工确认后进入当前项目的 `03-工作过程/AI协作`。

### 安全

- Provider、可执行文件、参数和环境变量均由服务端固定；使用 `shell:false`，Prompt 不拼接进命令行。
- 每次运行使用独立任务目录，全局最多两个活动任务；取消只回收当前进程组。子进程仍以本机用户身份运行，独立目录不等于操作系统沙箱。
- AI API 仅接受回环、同源和带 CSRF 标记的写请求，拒绝任意命令、路径和环境变量字段。
- 事件、错误、SQLite 与审计日志在落盘前脱敏，不向页面暴露工作区、可执行文件或原始环境。
- 导入器拒绝路径穿越、软链接、疑似凭证、可执行 HTML 和超限结果；索引校验失败时回滚新文件。
- 扩充 OpenAI/Anthropic、Google、AWS、GitHub 与 PEM 私钥识别；权威原文复制给 Agent 前再次扫描。
- SQLite 查询索引损坏时自动降级到 manifest 权威记录；导入崩溃恢复会重新执行索引刷新并支持安全重试。

## [0.2.0] - 2026-07-14

### 新增

- 驾驶舱名称、使用者、创作定位、增长计划、净增粉目标和项目日期可配置，并受控写回 Obsidian。
- 支持从六个已适配平台中配置 1—6 个账号，不要求每位使用者拥有相同账号组合。
- 新增可恢复的内容归档、制作完成事实和经过核验的平台发布记录。
- 内容复盘与账号拆解支持新建、编辑、确认和重新确认。
- 新增独立每日复盘，只有项目期内已确认的每日复盘才计入行动目标。
- 新增空 Vault 初始化脚本和脱敏配置模板，可为另一位使用者建立独立实例。
- 新增本地写入冲突检测、原子写入、备份、审计和索引失败回滚。
- 新增持续集成门禁，使用脱敏临时 Vault 验证安装、索引、测试和构建。

### 调整

- 首页改为“净增粉目标 → 行动目标 → 今日三件事”的决策顺序。
- 内容工作台收敛为选题、发布和复盘三个阶段，不再把制作过程当作资产状态。
- 行动目标完成数全部从有时间和证据的 V2 资产派生，不开放人工修改。
- Obsidian 外部修改通过统一事件通道即时回流，页面聚焦时补拉最新版本。
- 未初始化实例使用中性默认值；已有使用者的个性化配置由 Vault 设置文件承载。
- 今日任务使用受控 Markdown 区块更新，保留同一文件中的备注与其他检查清单。

### 安全

- 写接口仅接受本机回环和同源请求。
- 所有可写路径限制在明确白名单中，并拒绝绝对路径、路径穿越和软链接。
- 敏感内容和疑似凭证不会进入驾驶舱索引。
- 新 Vault 初始化中途失败时，会移除本轮已经创建的权威文件，避免留下半初始化状态。
- 新 Vault 初始化会逐段校验父目录，拒绝沿软链接或非目录节点把权威文件写出 Vault。
- 同一服务进程内的不同存储实例共享按文件串行队列，并发保存同一旧版本时只允许一个写入成功。

## [0.1.0] - 2026-07-13
