# v0.1.0 升级到 v0.2.0

## 已有使用者

已有 Vault 不运行 `setup:vault`。初始化脚本只服务于全新实例，遇到权威文件已存在会停止。

升级顺序：

1. 保留现有 Vault，并确认运行时状态目录的备份可用。
2. 更新代码后执行 `npm ci`。
3. 执行 `npm run index` 与 `npm run validate:data`，先验证现有资产能够通过新契约。
4. 打开驾驶舱设置，核对产品名、使用者、创作定位、增长计划、项目周期与净增粉目标；首次保存会创建 `99-系统/自媒体驾驶舱/驾驶舱设置.md`。
5. 核对行动目标。旧的 `confirmed_content_reviews` 读取时兼容为 `confirmed_daily_reviews`，下一次保存目标后写回新口径。
6. 内容复盘、账号拆解和每日复盘分别写入新的白名单目录，不会移动旧内容。

升级不会修改粉丝基线证据。平台账号注册表继续是当前粉丝的权威源，设置文件只引用既有项目目录和基线文件。

## 新使用者

新实例复制 `.env.example` 与 `setup.example.json`，必须替换 Vault 路径、使用者信息、`2000-01-01` 示例基线日期、账号数据和证据说明，再运行 `npm run setup:vault -- --config ./setup.local.json`。

初始化采用只创建不覆盖策略；若中途失败，本轮已经创建的权威文件会自动移除。空目录可以保留，不代表初始化成功。

## 回退边界

- 代码可退回 `v0.1.0`，但不要删除 v0.2.0 新建的 Obsidian 资产。
- 所有网页写回仍经过哈希冲突检查、原子替换、备份和失败回滚。
- 如果新索引校验失败，停止使用写接口，先恢复到上一个可验证的代码版本，再检查报错对应的 Vault 文件。
